
# brain_dynamics / 计划书（临时）— 动力学分析管线优化可执行规格 v0.1

> 目的：把当前 dynamics_pipeline 的“结论叙事”升级为 **可证伪、可复现、可量化验收** 的工程与科学一体化方案。
> 本文件为临时计划书：实现完成后，将关键内容合并回主文档（更新日志/项目规范说明书/使用说明/AGENTS）并删除本文件。

---

## 0. 背景：我们要回答的科学问题（交付导向）

我们要解释并尽量回答两类问题：

- **Q1（低维动力学）**：为何高维结构/高维状态空间中，长期动力学表现为低维（吸引子维度小、局部有效自由度少）？
- **Q2（近临界稳态）**：为何模型会自发稳定在近临界（edge-of-chaos / marginal stable）附近？是否存在“能量约束/结构谱/慢模态”层面的机制支撑？

当前已有证据链（H1–H5）在 `dynamics_pipeline/README.md` 中已定义，但存在若干“证据稳健性/术语边界/复现性”薄弱环节，本计划对其加固。

---

## 1. 总体原则（必须遵守）

1) **证据分级**：结构来源必须标注等级  
- A 级：response_matrix（因果/干预定义更强）  
- B 级：graph_cache 的结构连接（若存在明确物理含义）  
- C 级：FC（Pearson 相关，仅相关，不可直接解释为 Jacobian/因果）

2) **非线性指标优先**：混沌/临界性判据以 Rosenstein LLE 为主；线性方法（DMD）仅作为互补结构证据。

3) **所有核心结论必须给出：稳定性（跨 seed / 跨轨迹 / 跨窗口）、失败率、拟合质量、以及可复现元数据**。

## 2. P0 优先级任务（会影响结论可信度，必须先做）

### P0-1：D₂（相关维数）估计的鲁棒化（从“单轨迹一次”→“多轨迹/多窗口分布”）

#### 现状问题
Phase 3h 当前只对单条轨迹计算 D₂，极易因轨迹长度、瞬态、尺度区间选择导致不稳定或偶然“看起来低维/高维”。

#### 目标
把 D₂ 输出从标量升级为 **统计分布 + 质量报告 + 失败率**，并使其成为 H2 的可靠证据之一。

#### 实现要求（代码）
在 `dynamics_pipeline/pipeline.py` 的 Phase 3h（attractor_dimension）中：
- 不再只取 `ref_traj`，改为：
  - 抽样 `K_traj` 条轨迹（默认 K=10，若 n_init < K 则用全部）
  - 每条轨迹抽样 `M_windows` 个时间窗口（默认 M=5）
  - 每个窗口长度 `L_window`（默认 min(300, T - burnin)；不足则跳过）
- 对每个（traj, window）调用 `analysis.embedding_dimension.correlation_dimension(X)` 计算：
  - `D2`
  - `fit_r2`
  - （如果实现里能返回）拟合区间信息：`r_min/r_max` 或 `n_points` 等
- 记录失败（异常、NaN、r2 太低）的次数与原因分布。

建议新增配置键（`dynamics.attractor_dimension.d2`）：
```yaml
dynamics:
  attractor_dimension:
    enabled: true
    d2:
      enabled: true
      k_traj: 10
      m_windows: 5
      window_len: 300
      burnin_fraction: 0.1
      min_r2: 0.90
      seed: 42
```

#### 输出字段（results["attractor_dimension"]）
至少输出：
- `D2_mean`, `D2_std`
- `D2_median`, `D2_iqr`
- `D2_n_total`, `D2_n_valid`, `D2_fail_rate`
- `D2_r2_mean`, `D2_r2_min`
- `D2_values`（可选；若太大可只存前 N 或存 histogram）

#### 验收标准
- 不同随机 seed（轨迹/窗口抽样）下 `D2_mean` 波动可控（定义：相对误差 < 20% 或绝对误差 < 0.5，按数据量再调整）。
- `D2_fail_rate` 明确可见；当 fail_rate 高时，报告自动降级 H2 证据（见 P0-4 rubic）。

### P0-2：DMD 术语与证据边界修正（避免“DMD=真 Jacobian”的强表述）

#### 现状问题
代码与输出文件存在历史命名（jacobian_*），README 叙事中也容易被理解成“真正 Jacobian”。

#### 目标
把 DMD 定位为：**吸引子采样分布上的全局最优线性近似（经验线性化/Koopman 有限维投影）**，用于结构互补，不作为混沌判据。

#### 实现要求（文档 + 输出）
- 在 pipeline 报告中为 DMD 增加字段：
  - `dmd_interpretation_note`: “DMD operator A 是最小二乘线性近似，不等价于点态 Jacobian。”
- 对输出文件名若短期不改，报告里解释“历史命名原因”。

#### 验收标准
- README/报告中不再出现“DMD = Jacobian（强等价）”措辞。
- 在 Phase 6 一致性检查中，若 `nonlinearity_index` 大于阈值，则 DMD 相关结论自动标注“仅参考”。

---

### P0-3：把 Critical Slowing Down（CSD）接入 H3（临界性）证据链

#### 现状问题
H3 主要靠 λ₁≈0 + ρ≈1，属于相空间/线性化证据；缺少时间序列侧的独立旁证。

#### 目标
增加 CSD（方差、自相关、趋势检验）作为 H3 的并列证据，提升说服力与可解释性。

#### 实现要求（代码）
- 将 `brain_dynamics/analysis/critical_slowing_down.py` 接入：
  - Phase 3 增加 `critical_slowing_down` 子步骤（默认 enabled）
  - 或 Phase 6 汇总时计算并纳入 report（成本更低但可视化可能少一些）
- 建议新增配置：
```yaml
dynamics:
  critical_slowing_down:
    enabled: true
    window: 50
    step: 10
    regions: "global"   # 或 top-k variance regions
    compute_ar1: true
    compute_variance: true
    kendall_tau: true
```

#### 输出字段
- `csd_ar1_trend`, `csd_var_trend`（含 tau、p 值）
- `csd_ar1_mean`, `csd_var_mean`
- （可选）滑窗曲线保存为 npy/png

#### 验收标准
- 在 near-critical 设定/能量扫描实验中，CSD 指标与 λ₁/ρ 有一致性趋势（至少不矛盾）。

### P0-4：证据等级（response_matrix vs FC）与自动“结论降级”机制

#### 目标
报告必须明确当前使用的结构证据等级，并自动调整结论措辞/可信度标识。

#### 实现要求
- Phase 2 结构分析输出中加入：
  - `connectivity_source`: `response_matrix | graph_cache | fc`
  - `evidence_grade`: `A | B | C`
- Phase 6 报告中加入：
  - `confidence_flags`: 例如 `{"H1":"HIGH","H2":"MEDIUM","H3":"LOW"}`（可先简化）
- 当 `evidence_grade == C`（FC）时：
  - 禁止把谱半径/特征值解释为“动力学 Jacobian 的稳定性边界”
  - 只能说“相关结构/统计结构与低维现象相关”，并在报告里显式降级。

#### 验收标准
- 任意一次运行的 `pipeline_report.json` 都能看出证据等级，并且结论文本/标记随之变化。

## 3. P1 优先级任务（提升解释力：回答“为什么低维/为什么临界”）

### P1-1：增加“局部内禀维度”估计（回答低维不是 PCA 幻觉）

#### 目标
从“全局维度估计（D2/PCA）”升级到“局部切空间有效维度”的证据。

#### 建议实现（轻量）
- 新增 `analysis/intrinsic_dimension.py`：
  - TwoNN intrinsic dimension 或 local PCA slope（kNN 距离比法）
- 输出 `local_id_mean/std` 或随时间的分布摘要。
- 作为 H2 的第三类证据：**局部几何低维**。

#### 验收标准
- 在同一模型轨迹上，local intrinsic dimension 明显小于 N，并与 D2/PCA 有合理关系（D2 <= local_id <= PCA_n90 通常应成立或至少不离谱）。
