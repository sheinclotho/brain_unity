## 1. 项目概述

### 1.1 核心目标

TwinBrain是一个**基于多模态脑成像数据的数字孪生脑系统**，主要功能包括：
- 大脑活动建模与预测
- Unity 3D实时可视化
- 多模态数据整合（fMRI、EEG、DTI）
- 虚拟刺激模拟

### 1.2 技术栈

| 层级 | 技术 |
|------|------|
| **核心框架** | Python 3.8+ |
| **深度学习** | PyTorch 1.12+ |
| **神经网络架构** | 异构图神经网络 (Heterogeneous GNN) |
| **可视化引擎** | Unity 2019 LTS+ |
| **通信协议** | WebSocket (websockets库) |
| **数据格式** | JSON, OBJ, NIfTI, BIDS |
| **神经数据处理** | nibabel, nilearn, scipy |

##### A. `dynamic_hetero_gnn.py`

**异构图神经网络架构**:

```python
class DynamicHeteroGNN(nn.Module):
    def __init__(self, config):
        super().__init__()
        self.node_embeddings = nn.ModuleDict({
            'region': nn.Linear(in_features, hidden_dim),
            'network': nn.Linear(in_features, hidden_dim)
        })
        
        self.gnn_layers = nn.ModuleList([
            HeteroConv({
                ('region', 'connects', 'region'): GCNConv(...),
                ('region', 'belongs_to', 'network'): SAGEConv(...),
            }) for _ in range(num_layers)
        ])
        
        self.temporal_model = nn.LSTM(hidden_dim, hidden_dim)
        self.attention = MultiHeadAttention(hidden_dim, num_heads)
        self.predictor = nn.Linear(hidden_dim, out_features)
        
    def forward(self, graph_data, edge_index_dict, edge_attr_dict):
        # 1. 节点嵌入
        x_dict = {node_type: self.node_embeddings[node_type](x) 
                  for node_type, x in graph_data.items()}
        
        # 2. 图卷积
        for gnn_layer in self.gnn_layers:
            x_dict = gnn_layer(x_dict, edge_index_dict)
            x_dict = {k: F.relu(v) for k, v in x_dict.items()}
        
        # 3. 时序建模
        x_temporal, (h, c) = self.temporal_model(x_dict['region'])
        
        # 4. 注意力机制
        x_attended = self.attention(x_temporal)
        
        # 5. 预测
        output = self.predictor(x_attended)
        return output
```

##### B. `hetero_trainer_v2.py`

**训练循环**:
```python
class HeteroTrainerV2:
    def train(self, model, data_loader, optimizer, epochs):
        for epoch in range(epochs):
            for batch in data_loader:
                # Forward
                pred = model(batch)
                loss = self.compute_loss(pred, batch.y)
                
                # Backward
                optimizer.zero_grad()
                loss.backward()
                optimizer.step()
                
                # Logging
                self.log_metrics(epoch, loss, pred, batch.y)
                
    def compute_loss(self, pred, target):
        # 多任务损失
        recon_loss = F.mse_loss(pred['reconstruction'], target)
        pred_loss = F.mse_loss(pred['prediction'], target)
        reg_loss = self.l2_regularization()
        
        return recon_loss + 0.5 * pred_loss + 0.01 * reg_loss
```

```csharp
using UnityEngine;
using System.Collections.Generic;
using Newtonsoft.Json;

namespace TwinBrain
{
    public class BrainVisualization : MonoBehaviour
    {
        // 配置参数
        [SerializeField] private string jsonPath = "StreamingAssets/state";
        [SerializeField] private GameObject regionPrefab;
        [SerializeField] private float regionScale = 0.5f;
        [SerializeField] private float activityThreshold = 0.2f;
        
        // 数据容器
        private List<BrainStateData> brainStates;
        private Dictionary<int, GameObject> regionObjects;
        private int currentTimePoint = 0;
        
        void Start()
        {
            LoadBrainData();
            CreateRegionObjects();
            UpdateVisualization();
        }
        
        void Update()
        {
            if (Input.GetKeyDown(KeyCode.Space))
            {
                ToggleAnimation();
            }
            if (Input.GetKeyDown(KeyCode.R))
            {
                ReloadData();
            }
        }
        
        private void LoadBrainData()
        {
            // 从JSON文件加载数据
            string[] jsonFiles = Directory.GetFiles(jsonPath, "*.json");
            brainStates = new List<BrainStateData>();
            
            foreach (string file in jsonFiles)
            {
                string jsonText = File.ReadAllText(file);
                BrainStateData state = JsonConvert.DeserializeObject<BrainStateData>(jsonText);
                brainStates.Add(state);
            }
        }
        
        private void CreateRegionObjects()
        {
            regionObjects = new Dictionary<int, GameObject>();
            
            if (brainStates.Count > 0)
            {
                foreach (var region in brainStates[0].brain_state.regions)
                {
                    GameObject obj = Instantiate(regionPrefab, 
                        new Vector3(region.xyz[0], region.xyz[1], region.xyz[2]), 
                        Quaternion.identity);
                    obj.name = $"Region_{region.id}_{region.label}";
                    obj.transform.localScale = Vector3.one * regionScale;
                    regionObjects[region.id] = obj;
                }
            }
        }
        
        private void UpdateVisualization()
        {
            if (currentTimePoint >= brainStates.Count) return;
            
            var currentState = brainStates[currentTimePoint];
            
            foreach (var region in currentState.brain_state.regions)
            {
                if (regionObjects.ContainsKey(region.id))
                {
                    GameObject obj = regionObjects[region.id];
                    
                    // 根据活动值更新颜色
                    float activity = region.activity;
                    Color color = GetActivityColor(activity);
                    obj.GetComponent<Renderer>().material.color = color;
                    
                    // 根据活动值更新大小（可选）
                    if (activity > activityThreshold)
                    {
                        float scale = regionScale * (1 + activity);
                        obj.transform.localScale = Vector3.one * scale;
                    }
                }
            }
        }
        
        private Color GetActivityColor(float activity)
        {
            // 低活动=蓝色，高活动=红色
            return Color.Lerp(Color.blue, Color.red, activity);
        }
    }
}
```

## 5. 关键算法实现

### 5.1 球体网格生成算法

```python
def generate_sphere_vertices(center, radius, resolution):
    """
    使用球坐标系统生成球体网格
    
    球坐标公式:
        x = r * sin(θ) * cos(φ)
        y = r * sin(θ) * sin(φ)
        z = r * cos(θ)
    
    其中:
        θ (theta) = 纬度角 [0, π]
        φ (phi) = 经度角 [0, 2π]
    """
    vertices = []
    normals = []
    faces = []
    
    # 生成顶点
    for i in range(resolution + 1):
        lat = np.pi * i / resolution  # 纬度
        
        for j in range(resolution + 1):
            lon = 2 * np.pi * j / resolution  # 经度
            
            # 单位球面坐标
            x = np.sin(lat) * np.cos(lon)
            y = np.sin(lat) * np.sin(lon)
            z = np.cos(lat)
            
            # 缩放并平移
            vertex = center + radius * np.array([x, y, z])
            vertices.append(vertex)
            
            # 法线（指向球外）
            normals.append([x, y, z])
    
    # 生成面（四边形分为两个三角形）
    for i in range(resolution):
        for j in range(resolution):
            # 四个顶点索引
            v1 = i * (resolution + 1) + j
            v2 = v1 + 1
            v3 = (i + 1) * (resolution + 1) + j
            v4 = v3 + 1
            
            # 两个三角形
            faces.append((v1, v3, v2))
            faces.append((v2, v3, v4))
    
    return np.array(vertices), np.array(normals), faces
```

### 5.2 异构图消息传递

```python
def hetero_message_passing(x_dict, edge_index_dict, edge_attr_dict):
    """
    异构图神经网络的消息传递
    
    对于每种边类型 (src_type, edge_type, dst_type):
        1. 聚合邻居节点的特征
        2. 应用边类型特定的转换
        3. 更新目标节点特征
    """
    out_dict = {}
    
    for edge_type, edge_index in edge_index_dict.items():
        src_type, relation, dst_type = edge_type
        
        # 源节点特征
        x_src = x_dict[src_type]
        
        # 边特征（如果有）
        edge_attr = edge_attr_dict.get(edge_type, None)
        
        # 消息计算
        messages = compute_messages(x_src, edge_index, edge_attr)
        
        # 聚合
        aggregated = aggregate_messages(messages, edge_index[1])
        
        # 更新
        if dst_type not in out_dict:
            out_dict[dst_type] = []
        out_dict[dst_type].append(aggregated)
    
    # 合并来自不同边类型的消息
    for node_type in out_dict:
        out_dict[node_type] = sum(out_dict[node_type])
    
    return out_dict
```

### 5.3 多步预测算法

```python
def multi_step_prediction(model, initial_state, n_steps):
    """
    自回归多步预测
    
    t=0: initial_state
    t=1: model(initial_state)
    t=2: model(predicted_t1)
    ...
    t=n: model(predicted_t{n-1})
    """
    predictions = [initial_state]
    current_state = initial_state
    
    for step in range(n_steps):
        with torch.no_grad():
            # 预测下一步
            next_state = model(current_state)
            predictions.append(next_state)
            
            # 更新当前状态
            current_state = next_state
    
    return torch.stack(predictions, dim=1)  # [batch, n_steps+1, features]
```

## 13. 扰动分析框架：NPI 与 TwinBrain 对比及改进

> 本节记录对参考项目 NPI（Neural Perturbation Inference）框架的深入分析，
> 以及 TwinBrain 在此基础上的设计取舍和原创改进。

---

### 13.1 NPI 框架概述

**NPI（Neural Perturbation Inference）** 是一种利用神经网络代理模型推断脑区有效连接（Effective Connectivity, EC）的计算框架，核心思路来源于系统控制论：**若扰动系统的输入并观测输出变化，便可推断系统内部的因果结构**。

#### NPI 的三大核心功能

| 功能 | 函数 | 原理 |
|------|------|------|
| 训练代理模型 | `train_NN()` | 在经验数据上训练 MLP/CNN/RNN/VAR 做下一步预测 |
| 有限差分 EC 推断 | `model_EC()` | 对输入施加扰动 δ，测量输出变化 → EC[j,i] = Δout_i/δ |
| Jacobian EC 推断 | `model_Jacobian()` | 自动微分计算代理模型的 Jacobian 矩阵 |
| 模型 FC 模拟 | `model_FC()` | 用随机噪声驱动代理模型，计算功能连接 FC |

#### NPI 的代理模型选项

NPI 提供四种代理模型，适用不同假设：

| 模型 | 结构 | 适用假设 |
|------|------|---------|
| ANN_MLP | 多层感知机 | 无先验，通用非线性 |
| ANN_CNN | 1D 卷积 | 时间局部特征提取 |
| ANN_RNN | RNN + MLP编码器 | 显式时序依赖 |
| ANN_VAR | 线性全连接 | 向量自回归（线性EC） |

#### NPI 的 EC 推断原理（有限差分法）

```
对每个源区域 j（0..N-1）:
  1. 用全部输入样本计算基线预测：y_base = model(X)
  2. 在最近一步的区域 j 处加入扰动 δ：X_pert = X + Δ_j
  3. 计算扰动后预测：y_pert = model(X_pert)
  4. EC[j, :] = mean_t(y_pert - y_base) / δ

含义：EC[j, i] > 0 表示区域j的激活导致区域i的激活增强（兴奋性连接）
      EC[j, i] < 0 表示区域j的激活导致区域i的激活减弱（抑制性连接）
```

---

### 13.2 TwinBrain 当前扰动框架概述

TwinBrain 的扰动设计分布在三个层次：

#### 层次 A：波形注入（`model_server.py`）

```
用户选择刺激波形（正弦/脉冲/斜坡/持续）+ 目标区域
→ 生成刺激信号 stim[t]
→ 直接加到目标区域的当前状态值上
→ Wilson-Cowan 式动力学演化：
    excitation = tanh(state + stim*0.6 + connectivity@state*0.25)
    state = state*0.82 + excitation*0.18 + noise
```

#### 层次 B：空间扩散（`realtime_server.py` / `_demo_simulate`）

```
在 Fibonacci 球面坐标空间中：
→ 对每个目标区域计算高斯权重：w_j = exp(-d²_ij / 2σ²)，σ=30mm
→ 叠加到外部输入：stim_in[j] = amplitude * w_j
→ 偏差驱动漏积分器（以用户选定的 init_arr 为稳定平衡点）：
    deviation = state - init_arr          # 偏离基线的量
    net_dev   = W @ deviation             # 只传播偏差，防止全局饱和
    delta     = tanh(stim_in*2.0 + net_dev*0.35) * 0.04
    leak      = deviation * 0.10          # ~10步后自然回到 init_arr
    state     = clip(state + delta - leak + noise, 0, 1)

注：旧版使用 tanh(state + stim + net*0.25) 的 WC 公式，
在 L1 归一化连接矩阵下于 state≈0.70 处存在全局引力点，
导致所有区域收敛到黄色，刺激效果不可见。新版无此问题。
```

#### 层次 C：逆向优化（`stimulation_simulator.py`）

```
给定初始状态和目标状态：
→ 用 Adam 优化器学习 stim_amplitudes（每区域独立）
→ 损失 = MSE(stimulated, target) + 0.1*L1 + 0.01*L2
→ 输出：激活幅度超过阈值的区域列表
```

### 13.4 优劣比较

#### NPI 的优势
1. **科学严谨**：EC 矩阵有明确数学定义，可与 DTI 结构连接、Granger 因果比较。
2. **数据驱动**：代理模型从真实数据学习，不依赖人工先验（如 Wilson-Cowan 参数）。
3. **双向推断**：Jacobian 法一次前向传播即得全图 EC，比逐区域扰动快 N 倍。
4. **模型无关**：任何可微黑盒都可使用，包括 GNN。

#### NPI 的局限
1. **静态 EC**：仅捕获平均因果结构，不支持任务状态切换（如静息态 vs 任务态的 EC 变化）。
2. **线性近似**：有限差分在小 δ 下近似非线性，但大扰动下有偏差。
3. **扁平输入**：假设时序为 (T, N) 平铺矩阵，无法直接利用图结构（EEG 与 fMRI 的连接拓扑）。
4. **无临床参数**：无刺激波形、频率等实际 TMS/tDCS 参数，不直接对应治疗方案。

#### TwinBrain 的优势
1. **多模态图结构**：异构 GNN 同时编码 fMRI/EEG/DTI 节点和跨模态边，结构信息更丰富。
2. **临床对应**：正弦（tACS）、脉冲（TMS）、持续（tDCS）波形直接映射临床刺激类型。
3. **空间解剖学**：高斯扩散基于 MNI 坐标，物理意义明确。
4. **逆优化设计**：`design_inverse_stimulation()` 可为给定目标状态找到最优刺激方案，在闭环神经调控研究中有潜力。
5. **实时可视化**：WebSocket + Three.js 交互让非专业用户直观理解扰动效果。

#### TwinBrain 的原有局限（已部分改进）
1. ~~**无 EC 量化**~~：现已通过 `PerturbationAnalyzer` 补齐。
2. **随机扰动基线**：早期用 `torch.randn` 作为初始状态（现已修复为振荡器基线）。
3. **平坦注入**：直接修改状态值而非通过模型前向传播（现在无模型时使用振荡器，有模型时用模型）。

### 13.5 改进与创新：`PerturbationAnalyzer`（`unity_integration/perturbation_analyzer.py`）

基于上述分析，TwinBrain 引入了新模块 `PerturbationAnalyzer`，在保留自身优势的同时吸收 NPI 的核心科学价值。

#### 设计目标

1. **兼容 NPI 的两种方法**（有限差分 + Jacobian），让用户可选择精度/速度权衡。
2. **统一接口**：支持 MLP 代理（自动训练，无需 GNN）和 GNN 代理（有模型时直接插入）。
3. **靶点推荐**：基于 EC 矩阵自动建议最优刺激位点（影响力最强的源区域）。
4. **活动增量预测**：EC 矩阵 × 刺激向量 → 全脑活动变化预测，直接送入前端时间轴。

#### 三种 EC 推断模式

| 模式 | 方法 | 速度 | 精度 | 需要数据 |
|------|------|------|------|---------|
| `demo` | 基于距离的高斯 EC | 极快 | 低（近似） | 否 |
| `perturbation` | 有限差分（NPI 原方法） | 慢（N 次前向） | 中 | 是 |
| `jacobian` | 自动微分 Jacobian | 快（1 次/样本） | 高 | 是 |

#### 三种靶点推荐策略

```python
suggest_targets(ec_matrix, n_targets=5, strategy="outgoing")
# outgoing:  行和最大 → 影响其他区域最强（适合刺激目标）
# incoming:  列和最大 → 最容易被其他区域影响（适合监测目标）
# hub:       行和与列和的几何均值最大 → 全局枢纽（适合网络干预）
```

#### 与 NPI 的关键差异

| 方面 | NPI（原始） | TwinBrain PerturbationAnalyzer |
|------|-----------|-------------------------------|
| 代理模型 | 独立训练的 MLP/CNN/RNN/VAR | MLP 自动训练 or GNN 直接复用 |
| 输入格式 | 平铺 (N×lags,) 向量 | 与 NPI 相同（兼容）|
| EC 后处理 | 原始矩阵 | + 靶点排名 + 活动增量预测 |
| 与前端集成 | 无 | WebSocket → Three.js 连接线可视化 |
| 离线模式 | 无 | 有（基于 Fibonacci 坐标的演示 EC）|

### 13.7 深度分析：NPI 到底能预测什么？EC 矩阵可信度几何？

> 本节直接回答开发者常见的困惑：NPI 这么简洁，真的有预测能力吗？
> 它推断出的「脑区连接组」是什么，可靠吗？如何看待它与 TwinBrain 代理预测的关系？

---

#### 13.7.1 NPI 能实现的预测——一步预测 vs 轨迹预测

NPI 本身只做**一步（one-step-ahead）预测**：给定过去 n_lags 帧的活动，预测下一帧。这个任务由 MLP 代理模型完成，与时序预测（ARIMA、RNN）的本质完全相同。

```
输入:  [x_{t-n}, ..., x_{t-1}, x_t]  →  输出: x̂_{t+1}
```

代理模型的**一步预测精度**取决于数据质量，通常：

| 数据条件 | 一步预测 r² | 说明 |
|----------|------------|------|
| T > 1000 帧，r² > 0.6 | ~0.7–0.9 | 好（fMRI 静息态，长扫描）|
| T = 300–500 帧，r² > 0.4 | ~0.5–0.7 | 一般（常见）|
| T < 200 帧，过拟合严重 | ~0.2–0.4 | 差（数据不足）|

**多步轨迹预测**（TwinBrain 自回归展开）的精度**随步数指数衰减**，这是混沌系统的普遍特性（Lyapunov 指数）：

```
误差增长 ≈ e^{λ·k}，其中 λ > 0 为系统最大 Lyapunov 指数
k = 5 步   → 误差 × e^{5λ}     → 勉强可参考
k = 20 步  → 误差 × e^{20λ}    → 仅供定性参考
k = 60 步  → 误差基本等于随机猜测
```

**实践结论**：
- NPI 的代理模型**不能**做可靠的长期轨迹预测，这不是代码缺陷，而是数学上的基本限制。
- TwinBrain 展示的 60 帧预测轨迹应被理解为「基于数据动力学的趋势可视化」，不是精确预报。
- 若需要较可靠的 5–10 步预测，surrogate 必须满足 overfit_ratio < 5 且 r² > 0.5。

---

#### 13.7.2 NPI 推断出的「EC 矩阵」是什么？

EC（Effective Connectivity，有效连接）矩阵 **EC[j, i]** 的定义是：

> 区域 j 的过去活动对区域 i 的未来活动的**因果影响强度**，在控制了所有其他区域的条件下。

这本质上是**非线性 Granger 因果**的矩阵表示：

```
传统 Granger 因果（VAR）：
    EC[j,i] = ∂x_{i,t+1} / ∂x_{j,t}   （线性情形，精确）

NPI / TwinBrain（MLP 代理）：
    EC[j,i] ≈ mean_t [ model(X + δ·e_j) - model(X) ] / δ    （有限差分）
    EC[j,i] ≈ mean_x [ ∂f_i(x) / ∂x_j ]                   （Jacobian）
```

**与其他连接类型的本质区别**：

| 连接类型 | 测量的是什么 | 工具 | 因果性 |
|----------|------------|------|-------|
| **结构连接 (SC)** | 白质纤维束密度 | DTI/dMRI | 无（解剖学） |
| **功能连接 (FC)** | BOLD 信号的相关系数 | fMRI 相关矩阵 | 无（关联性，非因果）|
| **有效连接 (EC)** | 一个区域活动影响另一个的方向性 | DCM、Granger、**NPI** | **有（统计因果）** |
| **突触连接** | 实际神经元突触强度 | 光学成像、多电极 | 有（生理学） |

EC 矩阵的**正值**意味着兴奋性影响（j 激活 → i 激活），**负值**意味着抑制性影响（j 激活 → i 压抑）。

---

#### 13.7.3 EC 矩阵的可靠性——诚实的评估

NPI 的 EC 矩阵可靠性受以下因素严重制约：

**① 代理模型质量（最关键）**

```
EC 矩阵的误差上界 ∝ 代理模型的预测误差
→ 若代理模型 overfit_ratio > 10, 则 EC ≈ 噪声
→ 若代理模型 overfit_ratio < 3,  则 EC 相对可信
```

TwinBrain 在 `ec_result.fit_quality` 中暴露这个指标，前端显示可信度徽章。

**② 数据量**

| 训练帧数 | 可期待的 EC 可靠性 | 建议 |
|---------|-----------------|------|
| < 200 | 极低（参数/样本比 > 5:1） | 不要用于科学分析 |
| 200–500 | 低到中等 | 可作定性参考，误差大 |
| 500–1000 | 中等 | 网络大方向可信，具体值有偏差 |
| > 1000 | 较高 | 与 DCM/Granger 一致性 r ≈ 0.3–0.5 |

fMRI 的典型 TR=2s、扫描 10 分钟 = 300 帧，这正好落在「低到中等」区间——这是 NPI 方法的主要局限所在。

**③ 稳定性假设（非平稳性问题）**

NPI 假设整段数据的因果结构不变（平稳性）。实际上，fMRI 大脑的连接会随任务、状态、疲劳等随时变化。一个针对全程训练的 EC 矩阵只能反映**平均连接结构**，捕捉不到动态切换。

**④ 血流动力学混淆（fMRI 专有）**

fMRI 的 BOLD 信号是神经活动的**间接测量**，经过了血流动力学响应函数（HRF，时间常数 ~5s）的卷积。这意味着：
- 30ms 内的真实神经因果关系在 2s/帧的 fMRI 中无法分辨
- 血流动力学延迟差异会制造假因果（区域 B 的 HRF 比 A 慢 → 误认为 A→B 是因果）
- EEG 的时间分辨率更高（ms 级），但空间分辨率低，EC 矩阵的解剖意义不同

**⑤ 结构验证（黄金标准）**

真实的 EC 可靠性最终要通过与独立测量的对比来判断：

```
验证方法（可靠性递增顺序）：
1. 重测信度：同一被试两次扫描的 EC 相关性（r ≈ 0.3–0.5 为可接受）
2. 组内一致性：多名被试 EC 矩阵的 ICC（ICC > 0.4 为可接受）
3. 结构一致性：与 DTI 结构连接矩阵的相关（r ≈ 0.2–0.4 典型）
4. 任务调制：任务态 vs 静息态 EC 差异是否与激活分析一致
5. 临床预测：EC 矩阵能否预测认知指标或疾病状态（最终验证）
```

TwinBrain 目前没有实现上述任何一种验证，**这是最重要的未来工作**。

---

#### 13.7.4 TwinBrain 代理预测 vs NPI EC 推断——做的是两件不同的事

```
NPI EC 推断：
    问题：「在我已观测的脑动力学中，哪些区域对哪些区域有因果影响？」
    方法：训练代理模型 → 提取 Jacobian（代理的结构） → EC 矩阵
    局限：静态、平均、依赖代理质量、不能直接预测刺激效果

TwinBrain 代理轨迹预测（predict_trajectory）：
    问题：「如果我现在刺激某几个区域，大脑接下来会怎么演化？」
    方法：将刺激扰动注入 z-score 空间 → 代理自回归展开 → 未来帧序列
    局限：多步精度衰减、刺激注入方式是近似的、需要代理本身可靠

二者共享同一个代理模型，但目的不同：
    EC 矩阵 = 代理的「权重结构」（理解）
    轨迹预测 = 代理的「前向传播」（预测）
```

**关键区别**：EC 推断回答「有什么连接」，轨迹预测回答「会发生什么」。前者是分析工具，后者是模拟工具。两者都依赖代理质量，但轨迹预测的误差还会随步数累积。

---

#### 13.7.5 刺激扰动注入的正确理解

TwinBrain 的 `predict_trajectory` 将刺激建模为「在当前 z-score 空间中添加一个外部输入」：

```python
x[-N:] += stim_amp * stim_weights   # 单位：z-score 标准差
```

这等价于说：「如果大脑在此刻的状态被外部强制偏移了 stim_amp 个标准差（在空间分布 stim_weights 上），代理模型预测接下来会发生什么。」

**物理对应**：
- stim_amp = 0.1  → 相当于 0.1σ 的活动提升（非常轻微，类似 tDCS 亚阈值效应）
- stim_amp = 0.5  → 0.5σ（中等，类似单次 TMS 脉冲的直接效应）
- stim_amp = 1.0  → 1σ（强烈，接近高强度刺激的直接激活）
- stim_amp > 2.0  → 生理上不合理，代理模型会饱和并产生伪影

**注意**：stim_amp 这里是相对于 BOLD 信号 z-score 的变化量，而不是物理电流强度。临床参数（mA、V/m）到 BOLD z-score 的换算目前不在 TwinBrain 的模型范围内，需要专门的生物物理模型（如 SimNIBS）。

## 第 14 节 — EC 验证实现与大脑状态分析模块

### 14.1 EC 验证：从文档到实现

#### 14.1.1 三种已实现的验证方法

**`handle_validate_ec`** 端点在已训练代理模型的基础上运行三项验证，无需重新训练：

| 方法 | 实现 | 预期结果 |
|------|------|---------|
| **半分可靠性** | 将训练样本 `_input_X` 对半分，用同一代理模型分别计算 EC₁ 和 EC₂，报告 Pearson r | r > 0.5 = 高度可靠；< 0.1 = 数据不足 |
| **EC vs 解剖距离** | 计算 `|EC[i,j]|` 与 MNI 距离的 Pearson r（使用 Fibonacci 球坐标） | 预期负相关（r < −0.1）；正相关为异常 |
| **EC vs FC** | 比较 `|EC|` 与 Pearson 功能连接矩阵的相关性 | r < 0.4 表示 EC 提供了超越简单相关的因果信息 |

**关键设计决策**：半分可靠性不重新训练两个代理，而是用同一个已训练代理评估两个不同的输入子集。这避免了重新训练的计算开销，同时仍然测量 EC 对数据分区的稳定性。

#### 14.1.2 未来可实现的验证（需要多被试/多次扫描数据）

以下验证需要额外数据，目前 TwinBrain 未实现：

1. **重测信度**：同一被试两次扫描 → 计算 EC₁/EC₂ → Pearson r（目标 r > 0.3–0.5）
2. **被试间一致性 (ICC)**：N 名被试的 EC 矩阵 → 组内相关系数（目标 ICC > 0.4）
3. **结构-功能一致性**：EC 矩阵与 DTI 结构连接矩阵的相关（目标 r > 0.2–0.4）
4. **任务调制验证**：任务态 vs 静息态 EC 差异是否与 GLM 激活一致
5. **临床预测**：EC 特征 → 认知分数或疾病状态预测（外部效度的最终证明）

### 14.2 对照轨迹（Counterfactual）：同时观测刺激效果与自然演化

#### 14.2.1 设计目标

用户在时间 t 对大脑施加虚拟刺激，希望同时看到：
- **刺激轨迹**：从 t 开始，施加刺激后大脑的预测未来
- **对照轨迹**：从同一时间 t 开始，**不**施加刺激的自然演化预测

两条轨迹共享相同的初始状态，差异完全来自刺激。这是评估刺激效果的科学正确方法。

#### 14.2.2 实现机制（三条路径均支持）

| 仿真路径 | 刺激轨迹 | 对照轨迹 |
|---------|---------|---------|
| **Wilson-Cowan** | `_demo_simulate(amplitude=A)` | `_demo_simulate(amplitude=0)` |
| **代理 MLP** | `predict_trajectory(stim_fn)` | `predict_trajectory(lambda k: 0)` |
| **GNN ModelServer** | `simulate_stimulation(amplitude=A)` | `_demo_simulate(amplitude=0)` |

Wilson-Cowan 的两次调用使用相同的随机种子 `rng=np.random.default_rng(0)`，确保噪声序列完全相同，差异仅来自刺激。代理 MLP 本身无噪声（确定性 MLP），两次调用自然可比。

#### 14.2.3 前端交互

`simulation_result` 响应中新增 `counterfactual_frames` 字段。前端：
- 接收到 `counterfactual_frames` 后，时间轴底栏显示"○ 对照轨迹"切换按钮
- 点击切换：显示对照轨迹（"○ 对照"标签，橙色→蓝色）
- 再次点击：恢复刺激轨迹
- 两条轨迹帧数相同，时间轴滑块可用于任何位置对比

### 14.3 大脑状态分析模块（`BrainStateAnalyzer`）

#### 14.3.1 设计理念：为什么不做疾病分类？

疾病分类模型的根本局限：

```
分类模型要求:
  ① 大量标注数据（健康 vs 疾病，每类数百名被试）
  ② 稳定的疾病表型（而精神疾病和神经疾病在生物学上高度异质）
  ③ 可解释的特征（监管机构要求）
  ④ 可泛化到新医院/设备的分布外数据

TwinBrain 目前有:
  ① 单被试数据（或少数几名）
  ② 无标签
  ③ 无标准解剖对齐（Fibonacci 球，非真实 MNI）
  ④ 无跨中心验证
```

**结论**：做分类模型在目前的数据条件下是错误的研究设计，无论准确率多高都不可信。

#### 14.3.2 规范建模（Normative Modeling）——正确的框架

规范建模（Rutherford et al. 2022, Nat. Neurosci.）是当前计算精神病学的金标准：

```
核心思想：
  1. 在大型健康参考样本上建立"正常范围"（均值 ± 置信区间）
  2. 对每个新被试，计算其每个脑区的偏差 z 分数
     z_i = (x_i - μ_i) / σ_i
  3. z > 2.5 的区域被标记为"统计异常"——不是诊断，是信号
  4. 异常模式的组合（异常子图）与疾病亚型关联

关键优势：
  • 不需要疾病标签
  • 每个被试都有唯一的异常图谱（个体化）
  • 可量化不确定性（与分类不同）
  • 区域级别的可解释性
```

#### 14.3.3 已实现的无标签分析方法

**`handle_analyze_brain`** 端点，两种方法：

**方法 1：时段偏差图谱（`deviation`）**

```
输入: 已加载的时序数据 (T, 200)
操作: 
  window1 = ts[:T//2]   (前半段，作为参考)
  window2 = ts[T//2:]   (后半段，作为测试)
  z_i = |mean(window2[:,i]) - mean(window1[:,i])| / std(window1[:,i])
输出:
  • (200,) 归一化偏差图（颜色覆盖在 3D 脑区上）
  • 偏差最大的前 10 个脑区 ID（高亮显示）
  • 解读字符串（差异轻微/中等/显著）

应用场景:
  • 检查同一被试在不同任务阶段的脑区变化
  • 识别在扫描过程中"漂移"最多的脑区（可能是噪声 or 真实信号）
  • 对比两个实验条件（需要在 .pt 文件中有条件分割）
```

**方法 2：EC 枢纽分析（`graph_metrics`）**

```
输入: 已推断的 EC 矩阵
操作:
  hub_score_i = √(out_strength_i × in_strength_i)
  local_eff_i  = mean EC weight among region i's neighbours
  global_eff   = mean non-zero EC weight × network density
输出:
  • (200,) 归一化 hub 分数（颜色覆盖：红色 = 高度枢纽区域）
  • 前 10 个枢纽区域 ID
  • 全脑效率和连接密度数值

科学意义:
  Hub 区域是在神经网络中充当"中继站"的脑区，对信息整合至关重要。
  低效率 + 低密度 → 网络破碎（与老年化/神经退行性疾病关联）
  高密度 + 不均匀 hub 分布 → 可能存在小世界网络结构
```

#### 14.3.4 通用分析扩展路线图（未来实现）

```
已实现: 自比较（同一记录的两个时窗）
               ↓
规划中: 跨会话比较（两次独立录制的 EC/偏差对比）
               ↓
规划中: 规范参考库（多被试健康数据 → 建立均值 ± CI → 单被试偏差）
               ↓
未来:   时序异常检测（滑动窗口 EC → 检测连接切换事件）
```
