## [9.4.0] - 2026-02-26

### 修复 — 虚拟刺激第 0 帧二级静默失效：start_frame 指向错误帧

#### 根本原因（量化验证）
- **第一级修复（v9.3.0）**已将预刺激阶段从 10 个静态副本改为动态 WC 步进，并引入 `start_frame=10` 指向第一个刺激帧（stim onset, k=0）。
- **本次发现的第二级失效**：正弦包络在 k=0 时几乎为零（`sin(π×0.5/60)≈0.026`），WC 步进产生的 delta 仅约 **0.001**。数值审计表明在 `start_frame=10` 处：
  - 目标脑区相对 `init_arr` 的偏差：0.001（不可感知）
  - 不同目标脑区之间的最大可见差异：0.0015（不可感知）
  - 对比峰值帧（frame=40）：差异 0.28，为起始帧的 **193 倍**

#### 修复（`unity_integration/realtime_server.py` & `web_frontend/app.js`）
`start_frame` 改为**按刺激模式自适应**，指向累积效果最大的帧：

| 模式 | 新 start_frame | 峰值帧处可见偏差 | 改善倍数 |
|------|---------------|----------------|--------|
| sine | PRE + DUR//2 = 40 | 0.276 | 44× |
| pulse | PRE + min(9, DUR-1) = 19 | 0.142 | 6× |
| ramp | PRE + DUR*3//4 = 55 | 0.227 | 31× |
| constant | PRE + DUR//2 = 40 | 0.279 | 36× |

- WC 路径：`_peak_k` 计算后 `"start_frame": _PRE + _peak_k`
- 代理 MLP 路径：相同的 `_s_peak_k` 逻辑
- 离线前端：`simStartFrame = PRE + Math.floor(DUR / 2)`

## [9.3.0] - 2026-02-25

### 修复 — 虚拟刺激模块根本性错误

#### Bug 1：未选脑区时静默使用随机脑区（前端 `app.js`）
- **问题**：点击"施加刺激"时若未选中任何脑区，旧代码会随机选取 5 个区域，用户完全不知道刺激了哪里。
- **修复**：未选脑区时立即在 `#frame-info` 显示橙色警告"⚠ 请先点击脑区球体选择刺激目标（可多选）"，3 秒后消失，不发送请求。
- **不受影响**：EC 刺激推荐按钮（`btn-ec-stim`）会在触发刺激前自动填充 `selected`，逻辑不变。

#### Bug 2：WC 模型有 ~0.70 的全局引力点——所有脑区变黄（后端 `_demo_simulate`）
- **根因**：原 Wilson-Cowan 步进公式 `tanh(state + stim_in + net*0.25)` 在归一化连接矩阵下等价于 `tanh(1.25*state)`，固定点在 `state ≈ 0.70`（热图中对应黄色）。即使不施加任何刺激，经过 ~30 步后所有 200 个区域都收敛到黄色，刺激效果完全不可见。
- **修复**：以 `init_arr`（用户选定的当前大脑状态）为稳定平衡点，改用**偏差驱动漏积分器**：
  ```python
  deviation = state - init_arr       # 偏离基线的量
  net_dev   = W @ deviation          # 只传播偏差，不传播绝对活动
  delta     = tanh(stim_in*2.0 + net_dev*0.35) * 0.04
  leak      = deviation * 0.10       # ~10步后自然回到 init_arr
  state     = clip(state + delta - leak + noise, 0, 1)
  ```
- **效果验证**：修复前 80% 区域变黄；修复后仅 12% 区域高于 0.7，被刺激区域清晰上升，远端区域保持基线，刺激后逐步恢复。

## [9.2.0] - 2026-02-23

### 新增 — 显示模态切换卡片 & EEG 通道数指示

#### EEG 可视化决策
评估了四种 EEG 展示方案（2D 地形图、3D 电极、插值脑区、通道数指示），最终采用：
- **保留当前插值方案**（N通道→200脑区）作为主要可视化，符合 TwinBrain 的连接分析定位
- **新增通道数指示器**：模态信息显示 `EEG 64通道→200×384帧`，让用户清楚了解数据映射关系

#### 显示模态卡片重构（侧边栏右侧）
**之前**：模态切换埋在"加载缓存数据"卡片内，`display:none` 初始不可见，用户不知道该功能存在

**之后**：
- 独立的"显示模态"卡片，**永久可见**，不再隐藏
- 初始状态：fMRI / EEG 两个按钮均 disabled（灰显），提示"加载 .pt 文件后可切换"
- 加载数据后：仅有数据的模态按钮启用（只有EEG→fMRI 按钮不可点；反之同理；两者都有→都可切换）
- 按钮信息描述原始通道/区域数：`EEG 64通道→200×384帧`、`fMRI 200区×384帧`

#### EC 可信度徽章（前端）
- `handleECResult` 读取 `fit_quality.reliable`
- 可靠时显示绿色 `✓ 可靠`，过拟合时显示橙色 `⚠ 过拟合 260×`

#### 后端 `handle_load_cache` 新增字段
- `n_fmri_regions`：加载的 fMRI 实际区域数（通常 200，直接映射无插值）
- `n_eeg_channels`：加载的 EEG 实际通道数（如 64/128，插值到 200 槽位）

## [9.1.0] - 2026-02-23

### 修复 — OMP 崩溃 & MLP 代理过拟合

#### OMP: Error #15 修复
- 在 `start.py` 和 `perturbation_analyzer.py` 最前面设置 `KMP_DUPLICATE_LIB_OK=TRUE`，
  避免 PyTorch + NumPy/MKL 双重加载 OpenMP 运行时导致的崩溃（在 Windows 上表现为 `libiomp5md.dll already initialized`）。

#### MLP 代理过拟合彻底修复
- **问题**: 384帧数据上验证MSE是训练MSE的260倍（0.62 vs 0.002），代理模型失效，EC推断结果不可信。
- **自适应容量**: 对 ≤500 样本的数据集按比例缩减网络宽度，防止参数过多。
- **Dropout（p=0.3）**: `_SurrogateMLP` 每隐层后添加 Dropout，提升泛化能力。
- **Weight Decay（1e-4）**: Adam 优化器添加 L2 正则化（对应 NPI 原版 `l2` 参数）。
- **早停（patience=12）**: 验证 MSE 不改善则提前结束并恢复最优权重，修复后 overfit ratio 降至 ~1.1×。

#### EC 响应质量增强
- `ec_result` 新增 `fit_quality` 字段 `{train_mse, val_mse, overfit_ratio, reliable, n_epochs}`，前端可显示可信度徽章。
- `activity_delta` 从仅用 top-1 源改为 top-3 源，预测更全面。
- 服务端缓存已训练代理（`_ec_analyzer_cache`），切换 jacobian↔perturbation 无需重新训练。

## [9.0.0] - 2026-02-21

### 新增 — NPI 启发的有效连接推断（EC Inference）

#### 背景

参考项目 `NPI.py` 实现了**神经扰动推断（Neural Perturbation Inference）**框架，
通过训练代理模型并对其施加扰动来量化脑区因果影响力（有效连接 EC）。
在深入分析两个框架的异同后，TwinBrain 引入了以下改进。

#### 新增文件

- **`unity_integration/perturbation_analyzer.py`**  
  `PerturbationAnalyzer` 类，实现三种 EC 推断模式：
  - `demo`：基于 Fibonacci 坐标的距离 EC（无需数据，毫秒级）
  - `perturbation`：NPI 原版有限差分法（训练 MLP 代理 + 逐区域扰动）
  - `jacobian`：自动微分 Jacobian 法（比有限差分快 N 倍，精度更高）
  
  同时提供靶点推荐（`suggest_targets`，三种策略：传出强度/传入强度/枢纽度）
  和活动增量预测（`predict_activity_delta`，EC 矩阵的线性近似）。

- **`NPI.py`**（来自 main 分支，参考实现，不修改）

#### 后端扩展（`realtime_server.py`）

新增 WebSocket 消息类型：
- **请求** `{type: "infer_ec", method: ..., n_lags: 5}`
- **响应** `{type: "ec_result", ec_flat: [40000], top_sources: [10], top_targets: [10], activity_delta: [200]}`

自动检测数据来源：
1. 若提供 `path` 字段 → 从 `.pt` 缓存加载时序
2. 若自动检测到缓存文件 → 使用
3. 否则 → 用 7 网络振荡器生成合成时序（演示用）

#### 前端扩展（`index.html` + `app.js`）

右侧面板新增「有效连接推断 (EC)」卡片：
- 下拉选择推断方法（Jacobian/有限差分/演示）
- **推断有效连接** → 触发 EC 推断，完成后在 3D 视图绘制彩色连接线
  （金色线 = 强连接，白色高亮 = 源区域，青色高亮 = 目标区域）
- **刺激推荐靶点** → 自动选中 EC 影响力最强的 5 个区域并触发刺激
- **隐藏连接线** → 清除可视化
- 离线演示模式：后端未连接时也可在浏览器内计算距离 EC 并可视化

#### 文档更新（`项目规范说明书.md` 第 13 节）

新增「扰动分析框架：NPI 与 TwinBrain 对比及改进」，内容包括：
- NPI 框架详解（三大功能 + 四种代理模型）
- TwinBrain 三层扰动架构（波形注入/空间扩散/逆向优化）
- 详细对比表（异同、优劣）
- `PerturbationAnalyzer` 设计文档
- 未来五个可探索方向

- `unity_one_click_install.py`：后续步骤提示包含 NativeWebSocket 安装说明。

### 根本性修复（HTTP 轮询 → 真正 WebSocket）

旧版所有 C# 脚本使用 `UnityWebRequest` 向 `http://localhost:8765` 发 GET 请求，但后端是 WebSocket 服务器。HTTP GET 请求**不能接收服务器推送消息**，导致：
- 每次只能拉取一次数据，无法实时更新
- 刺激结果、时间轴帧无法自动推送到客户端

新版通过 NativeWebSocket 建立持久化双向 WebSocket 连接，所有问题根治。
