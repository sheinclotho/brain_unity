## [6.0.0] - 2026-02-20

### 新增 — 3D 大脑可视化 + 完整数据流重构

- **Three.js 3D 脑区可视化**：200 个皮质区域以 Fibonacci 球面算法生成解剖学近似位置（MNI 空间），按脑网络分色，替代原来的平面蓝圆。玻璃脑轮廓框架提供直观深度感。
- **轨道相机控制**：鼠标拖动旋转、滚轮缩放，支持触屏。
- **时间轴滑块**：可拖动播放/暂停，显示当前帧 / 总帧数。
- **从 .pt 缓存文件加载时序数据**：新增 `load_cache` WebSocket 消息类型，读取 `hetero_graphs.pt` / `eeg_data.pt` 中的 `fmri.x_seq` 时间序列，自动转为归一化 [0,1] 活动数组并推送给前端。
- **耦合振荡器刺激仿真**：`ModelServer.simulate_stimulation()` 改用 Wilson–Cowan 式耦合振荡器（局部连接 + 同侧镜像连接），物理上更合理，空间传播效应真实可见。

### 修复

- **WebSocket 消息类型匹配**：前端发送 `get_brain_state` / `simulate_stimulation`，后端现在自动识别并路由到正确处理器（原来类型名不一致导致所有请求失败）。
- **响应格式标准化**：所有响应统一包含 `activity: [200 floats]` 平铺数组，前端直接使用，无需多层嵌套取值。
- **仿真结果改为 `simulation_result` 类型**，携带完整 `frames` 数组，前端加载时间轴后自动播放。
- **asyncio 事件循环兼容**：`start.py` 调用 `start_async()` 而非 `start()`，修复启动时崩溃。
- **`_state_to_json`** 现在同时输出 `"activity"` 平铺字段，方便多处消费。

### 架构变更

- 前端彻底重写（`web_frontend/index.html` + `app.js`）：Three.js 替代原 2D Canvas。
- 后端新增方法：`handle_load_cache`, `_find_cache_file`, `_find_hetero_data`, `_extract_time_series`, `_extract_activity_array`, `_get_connectivity_matrix`。

### 待优化

- [ ] 使用真实 Schaefer-200 atlas MNI 坐标替代程序生成的近似坐标
- [ ] 当训练好的模型已加载时，刺激仿真真正经过 GNN forward pass
- [ ] 支持 NIfTI (.nii.gz) 格式的原始 fMRI 文件直接加载
- [ ] 添加脑连接矩阵叠加显示（结构/功能连接弦图）
- [ ] 单元测试覆盖率 ≥ 60%

## [5.0.0] - 2026-02-20

### 新增
- 新增零配置启动脚本 `start.py`，直接运行即可使用，无需任何参数
- 新增 Web 前端 (`web_frontend/`)，支持在浏览器中直接可视化大脑活动，无需安装 Unity
- Web 前端支持：点击脑区施加虚拟刺激、实时显示活动变化、颜色热图
- 新增 `AGENTS.md`，为 AI 编程助手提供协作规范和错误日志
- 新增 `使用说明.md`，面向非技术用户的简明操作指南

### 变更
- **文档整合**：将原有 9 个 Markdown 文件整合为 4 个（规范、使用说明、更新日志、AI 提示词）
- `start.py` 替代原有复杂的 `unity_startup.py` 作为默认启动入口（`unity_startup.py` 仍保留供高级用户使用）
- 默认所有模块启用，无需手动选择
- WebSocket 默认绑定 `127.0.0.1`（本地），提升安全性

### 待优化
- [ ] Unity 3D 场景自动化设置流程仍需手动操作，有待简化
- [ ] Web 前端目前使用简化的球形脑区，待集成真实 FreeSurfer 脑表面数据
- [ ] 添加单元测试覆盖率至 60% 以上
- [ ] 支持 Docker 一键部署
