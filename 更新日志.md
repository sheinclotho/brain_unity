## [2026-03-08] Lyapunov 分析 Bug 修复 & dynamics 模块综合审查（第二轮）

### 新增 Bug 修复

**Bug 4（Critical）: `skipped_wolf` 强制 stable 即使 LLE > 0**
- 根因：混沌吸引子吸引效应使 distance_ratio < threshold（所有轨迹汇入同一混沌吸引子），convergence-first 误触发，强制 regime="stable"，但 Rosenstein LLE 实际 > 0。
- 修复：在 skipped_wolf 分支检查 `primary_mean < 0`：LLE < 0 → 覆盖 stable；LLE ≥ 0 → 保留原始 LLE 分类并发出警告。
- 文件：`twinbrain-dynamics/analysis/lyapunov.py`

**Bug 5（Critical）: `lyapunov_spectrum_wolf` 上下文稀释 → 所有 λ 趋同为 +0.13**
- 根因：每周期对 baseline + k 扰动方向调用 `rollout()`，每次重置 base_graph 上下文。k 个方向全部被相同上下文稀释因子影响，λ₁≈λ₂≈...≈λₖ（强混沌假象）。
- 修复：改用 `wolf_rollout_pair()` + 推进的 `wolf_context`，与 scalar Wolf 方法保持一致。回退路径（legacy simulator）保留旧行为。
- 文件：`twinbrain-dynamics/analysis/lyapunov.py`

### 新增改进

**初始多样性歧义警告（Attractor Ambiguity）**
- 新增 `attractor_ambiguous` 检测区间（0.020 ≤ initial_std < 0.087）：
  发出警告说明 distance_ratio 下降可能是"汇入混沌吸引子"，而非"固定点收敛"。

**跨步骤一致性检查（Step 9 vs Step 15）**
- Step 15 完成后自动对比 Step 9（Lyapunov LLE）和 Step 15（Lyapunov 谱）的分类结果。
- 若 Step 9="stable" 但 Step 15 有 n_positive>0 个正指数 → 打印醒目 WARNING 方框。
- 文件：`twinbrain-dynamics/run_dynamics_analysis.py`

## [2026-03-08] Lyapunov 分析 Bug 修复 & dynamics 模块综合审查

### Bug 修复

**Bug 1（Critical）: convergence-first 错误地将 `rosenstein` → `ftle`**
- 根因：`run_lyapunov_analysis` 的收敛优先逻辑无论原始方法是什么都切换为 FTLE，导致 TwinBrain 的 Rosenstein 正确估计（λ=-0.028）被错误的 FTLE 近零值（λ≈0.0002）替换。
- 修复：仅当 `method='wolf'` 或 `'both'` 时才切换为 FTLE。`method='rosenstein'`（默认）遇到收敛检测时保持不变，`skipped_wolf=True` 直接触发 "stable" 分类覆盖。
- 文件：`twinbrain-dynamics/analysis/lyapunov.py`

**Bug 2（Medium）: FTLE 数值底层导致收敛系统 λ≈0**
- 根因：收敛系统的 `dist(t)` 跌落到数值底层（1e-15）后，回归范围被平坦段主导，斜率≈0。
- 修复：`ftle_lyapunov` 增加底层感知回归——仅对 `log_dist > log(1e-12)` 的有信号区域做线性拟合；整段在底层时回退至预跳过段或返回 NaN。
- 文件：`twinbrain-dynamics/analysis/lyapunov.py`

### 性能优化

**`_autocorrelation` FFT 优化**
- 问题：O(T²) Python 循环，与已有 FFT ACF 实现重复。
- 修复：改为 FFT（O(T log T)），数值等价性验证 max_err=0。
- 文件：`twinbrain-dynamics/analysis/stability_analysis.py`

### 文档更新

- `n_segments=3` 配置项添加注释：说明仅对 Rosenstein 有价值，Wolf/FTLE 应为 1。
- `ftle_lyapunov` docstring 补充上下文稀释注意事项和底层效应说明。
- 收敛优先策略 docstring 更新以反映修复后的逻辑。
- AGENTS.md 新增根因分析和规则记录。

## [2026-03-07] spectral_dynamics 模块 — 连接组谱结构→低维动力学→近临界假设链验证

### 新增模块：`spectral_dynamics/`

实现 9 个独立实验，系统性验证三个核心假设：

```
H1  connectome 的谱结构是低秩的（少数特征值主导）
H2  网络动力学主要由少数特征模控制
H3  谱结构使系统自然接近临界（g≈1 时处于边缘混沌）
```

#### 模块结构

| 文件 | 对应实验 | 核心功能 |
|------|---------|---------|
| `a_connectivity_visualization.py` | A1/A2 | 原始连接矩阵热图；按社区排序后热图 |
| `c_community_structure.py` | C1-C3 | 谱聚类/Louvain 社区检测；模块度 Q；社区规模直方图 |
| `d_hierarchical_structure.py` | D1/D2 | Ward 层级聚类；Dendrogram；层级陡峭度指数 |
| `e1_spectral_analysis.py` | B/E1 | 特征值分布；谱有效维度（参与率 PR）；主导模式数 |
| `e2_e3_modal_projection.py` | E2/E3 | SVD 模态投影；模态能量分布；累积 80%/90%/95% 能量所需模态数 |
| `e4_structural_perturbation.py` | E4 | 边权重随机化/保度重连/低秩截断三种扰动；对比谱指标与 LLE |
| `e5_phase_diagram.py` | E5 | WC 耦合强度 g∈[0.1,3.0] 相图；Rosenstein LLE vs g；振荡幅度 vs g |
| `e6_random_comparison.py` | E6 | ER/保度/权重混洗随机网络谱比较；PR z-score 统计检验 |
| `f_pca_attractor.py` | F1/F2 | PCA 解释方差曲线；PC1-PC2 2D/3D 吸引子投影 |
| `compute_connectivity.py` | 数据加载 | 从 twinbrain-dynamics 输出加载轨迹/响应矩阵；计算 FC |
| `run_all.py` | 主入口 | CLI 一键运行全部或指定实验；生成 run_summary.json |

#### 设计原则（科学批判性实现）

1. **谱分析使用双矩阵**：有效连接矩阵（响应矩阵 R，来自 twinbrain-dynamics 步骤 6）
   和功能连接矩阵（FC，从轨迹计算），两者给出互补视角。
2. **模态投影使用 SVD**（非直接特征向量求逆）：对非对称 EC 矩阵数值稳定；
   显式处理 tanh 非线性使得 WC 轨迹在少数奇异模方向上能量集中不如线性系统强的情况。
3. **社区检测双算法**：优先 Louvain（若 networkx/community 可用），
   否则用确定性谱聚类（sklearn）+ 模块度最大化搜索最优 k。
4. **E5 相图在 WC 框架**而非直接修改 GNN 权重（后者需重新训练）；
   明确说明 WC 临界边界约 ρ≈1.5（非线性压缩效应）。
5. **E4 扰动在连接矩阵层面**而非图缓存层面；
   LLE 通过 twinbrain-dynamics 的 `rosenstein_lyapunov` 估计（若可导入），
   否则使用内置简化 Wolf 算法。
6. **F 模块 PCA** 是 FNN/D₂ 的线性补充；代码中明确标注其低估非线性维度的局限性。

#### 测试覆盖

- 全部使用合成数据（N=20），运行时间约 22 秒

## [9.9.0] - 2026-03-06

### 批判性科学审查 + 三大创新模块（twinbrain-dynamics v2.0）

#### 1. 新增 Transfer Entropy 信息流分析（）

**动机**：响应矩阵只能度量外部刺激的传播；无法捕获系统**自发运行时**的有向信息传递结构。
Transfer Entropy（Schreiber 2000）提供模型无关的有向功能连接估计：

4370TE(X 	o Y) = H(Y_{t+1}|Y_t^{(k)}) - H(Y_{t+1}|Y_t^{(k)},X_t^{(l)})4370

实现：等宽分箱频率估计量，速度比 KSG k-NN 快约 100×。
输出：TE 矩阵、净信息流矩阵、信息流中心性指标（广播枢纽、整合枢纽、净驱动区）。

#### 2. 新增网络可控性分析（）

**动机**：基于线性控制理论（Gu et al. 2015 Nat Commun），从响应矩阵推导最优刺激靶点。
三种互补指标：
- **平均可控性 AC**：以小能量驱动到大量状态（广播型枢纽，如默认模式网络）
- **模态可控性 MC**：驱动难以到达的高能模式（状态切换枢纽，如初级感觉皮层）
- **边界可控性 BC**：协调社区间通信（胼胝体型节点）

科学预期：AC-MC 应负相关（Gu 2015 图 2），即广播器不能同时是高效状态切换器。

#### 3. 新增临界慢化早期预警信号分析（）

**动机**：当动力系统临近分岔（critical transition）时，方差增大、自相关升高——
即临界慢化（CSD，Scheffer et al. 2009 Nature）。检测 CSD 可评估模型动力学的稳定性：
稳定模型的 EWS 评分应接近 0；EWS > 0.5 提示分岔风险。

实现指标：滑动方差/AR(1) 的 Kendall τ 趋势、去趋势波动分析（DFA）标度指数 α、偏度/峰度。
综合 EWS 评分（0-1），5 项指标各贡献 0.2 分。

## [9.8.0] - 2026-03-06

### 动力系统分析模块科学化升级（twinbrain-dynamics）

#### 1. 稳定性分析 — 方法 C 自适应相对阈值

**问题**：原方法 A/B 使用绝对阈值（delay_mean < 0.1 等），导致 n_regions=190 的真实模型
出现 100% "不稳定" 错误分类（延迟距离的 L2 范数随 sqrt(n_regions) 线性增长，绝对阈值失效）。

**新增方法 C（`classify_dynamics_adaptive`）**：
- `delta_ratio = delay_mean / traj_rms`（无量纲，n_regions 无关）
- `cv_delta = std / mean` 延迟序列的变异系数（区分规则/混沌）
- `spectral_peak_ratio`：延迟序列 FFT 主频功率占比（识别极限环）
- 分类规则：delta_ratio < 0.01 AND cv < 0.5 → 不动点；delta_ratio < 0.05 AND spectral > 0.4 → 极限环；delta_ratio < 0.15 → 亚稳态；else → 不稳定
- JSON 新增 `classification_counts_v2`（方法B）、`delta_ratio_stats`（分布统计）

**参考**：Eckmann & Ruelle (1985) Rev. Mod. Phys. 57:617; Kantz & Schreiber (1997)

#### 2. Lyapunov 分析 — Rosenstein 方法、多方向 FTLE、多段采样

**问题（Wolf 上下文稀释偏差）**：
在 TwinBrainDigitalTwin 模式下，Wolf 扰动仅施加于上下文窗口最后一步，被注意力机制稀释，
导致所有轨迹的 Wolf LLE 几乎相同（std ≈ 0.00006）——这不是系统真实 LLE。
FTLE（-0.005）与 Wolf（-0.057）差异 >10× 是上下文稀释偏差的诊断标志。

**新增 `rosenstein_lyapunov`（Rosenstein et al. 1993）**：
- 直接从轨迹数据估计 LLE，无需重新运行模拟器
- 不受上下文稀释偏差影响（在观测空间度量，非上下文空间）
- 为 TwinBrainDigitalTwin 模式推荐的主要 LLE 估计方法

**新增 `multi_direction_ftle`**：
- K 个随机扰动方向的 FTLE 集成平均，减少单方向估计的方差

**新增 `run_lyapunov_analysis` 参数**：
- `method="rosenstein"` 或 `"both"`（Wolf + FTLE + Rosenstein 交叉验证）
- `n_segments`：多段采样（默认 3），从轨迹不同位置采样 LLE，探索吸引子不同区域
- `rosenstein_max_lag`、`rosenstein_min_sep`：Rosenstein 方法参数
- **Wolf 偏差自动检测**：std < 1e-3 且 n_traj > 10 时发出警告，提示可能存在 twin mode 偏差
- `wolf_bias_warning` 标志加入结果字典和 JSON 报告

**默认配置更新**：
- `lyapunov.method = "both"（含 Rosenstein）`
- `lyapunov.n_segments = 3`

### 修复 — 离线前端 `stepLocal` 使用错误的 WC 吸引子公式

#### 根本原因（量化验证）
离线前端 `stepLocal` 函数（WebSocket 断开时的简化 WC 预览）使用旧公式：
```javascript
// 旧公式（错误）
const exc = Math.tanh(v + stimIn[i]);
return v * 0.85 + exc * 0.15;
```
该公式的稳定不动点为 `v* = tanh(v*)` → `v* = 0`（黑色）。与服务端已修复的偏差驱动漏积分器不一致。

**测量到的错误**：

| 阶段 | 旧公式（错误） | 新公式（正确） |
|------|--------------|--------------|
| 预刺激漂移（5步无刺激） | **max 0.065**（向黑色漂移） | **0.000**（精确保持在 act0） |
| 刺激后非靶区偏差 | **0.0064**（向 0 漂移） | **0.0000**（不动） |
| 刺激后恢复方向 | 向 0 衰减（错误） | 向 act0 衰减（正确） |

#### 修复（`web_frontend/app.js`）
```javascript
// 新公式（正确）—— 与 _demo_simulate 服务端 WC 公式一致
function stepLocal(state, stimIn) {
  return state.map((v, i) => {
    const delta = Math.tanh(stimIn[i] * 2.0) * 0.04;   // 外部驱动响应
    const leak  = (v - act0[i]) * 0.10;                 // 漏电（收敛到 act0）
    return Math.max(0, Math.min(1, v + delta - leak));
  });
}
```
- 无刺激 → v 精确锁定在 `act0`（预刺激和后刺激均不漂移）
- 有刺激 → 靶区显著上升（峰值偏差 0.169），非靶区保持不动
- 函数通过闭包访问 `act0`（定义在同一块中，无需修改调用方签名）

### 修复 — 虚拟刺激模块根本性错误

#### Bug 1：未选脑区时静默使用随机脑区（前端 `app.js`）
- **问题**：点击"施加刺激"时若未选中任何脑区，旧代码会随机选取 5 个区域，用户完全不知道刺激了哪里。

#### Bug 2：WC 模型有 ~0.70 的全局引力点——所有脑区变黄（后端 `_demo_simulate`）
- **根因**：原 Wilson-Cowan 步进公式 `tanh(state + stim_in + net*0.25)` 在归一化连接矩阵下等价于 `tanh(1.25*state)`，固定点在 `state ≈ 0.70`（热图中对应黄色）。即使不施加任何刺激，经过 ~30 步后所有 200 个区域都收敛到黄色，刺激效果完全不可见。
- **修复**：以 `init_arr`（用户选定的当前大脑状态）为稳定平衡点，改用**偏差驱动漏积分器**：
  ```python
  deviation = state - init_arr       # 偏离基线的量
  net_dev   = W @ deviation          # 只传播偏差，不传播绝对活动
  delta     = tanh(stim_in*2.0 + net_dev*0.35) * 0.04
  leak      = deviation * 0.10       # ~10步后自然回到 init_arr
  state     = clip(state + delta - leak + noise, 0, 1)
  ```
- **效果验证**：修复前 80% 区域变黄；修复后仅 12% 区域高于 0.7，被刺激区域清晰上升，远端区域保持基线，刺激后逐步恢复。

参考项目 `NPI.py` 实现了**神经扰动推断（Neural Perturbation Inference）**框架，
通过训练代理模型并对其施加扰动来量化脑区因果影响力（有效连接 EC）。
在深入分析两个框架的异同后，TwinBrain 引入了以下改进。

#### 新增文件

- **`unity_integration/perturbation_analyzer.py`**  
  `PerturbationAnalyzer` 类，实现三种 EC 推断模式：
  - `demo`：基于 Fibonacci 坐标的距离 EC（无需数据，毫秒级）
  - `perturbation`：NPI 原版有限差分法（训练 MLP 代理 + 逐区域扰动）
  - `jacobian`：自动微分 Jacobian 法（比有限差分快 N 倍，精度更高）
  
  同时提供靶点推荐（`suggest_targets`，三种策略：传出强度/传入强度/枢纽度）
  和活动增量预测（`predict_activity_delta`，EC 矩阵的线性近似）。

- **`NPI.py`**（来自 main 分支，参考实现，不修改）

#### 后端扩展（`realtime_server.py`）

新增 WebSocket 消息类型：
- **请求** `{type: "infer_ec", method: ..., n_lags: 5}`
- **响应** `{type: "ec_result", ec_flat: [40000], top_sources: [10], top_targets: [10], activity_delta: [200]}`

自动检测数据来源：
1. 若提供 `path` 字段 → 从 `.pt` 缓存加载时序
2. 若自动检测到缓存文件 → 使用
3. 否则 → 用 7 网络振荡器生成合成时序（演示用）

#### 前端扩展（`index.html` + `app.js`）

右侧面板新增「有效连接推断 (EC)」卡片：
- 下拉选择推断方法（Jacobian/有限差分/演示）
- **推断有效连接** → 触发 EC 推断，完成后在 3D 视图绘制彩色连接线
  （金色线 = 强连接，白色高亮 = 源区域，青色高亮 = 目标区域）
- **刺激推荐靶点** → 自动选中 EC 影响力最强的 5 个区域并触发刺激
- **隐藏连接线** → 清除可视化
- 离线演示模式：后端未连接时也可在浏览器内计算距离 EC 并可视化

#### 文档更新（`项目规范说明书.md` 第 13 节）

新增「扰动分析框架：NPI 与 TwinBrain 对比及改进」，内容包括：
- NPI 框架详解（三大功能 + 四种代理模型）
- TwinBrain 三层扰动架构（波形注入/空间扩散/逆向优化）
- 详细对比表（异同、优劣）
- `PerturbationAnalyzer` 设计文档
- 未来五个可探索方向

### 根本性修复（HTTP 轮询 → 真正 WebSocket）

旧版所有 C# 脚本使用 `UnityWebRequest` 向 `http://localhost:8765` 发 GET 请求，但后端是 WebSocket 服务器。HTTP GET 请求**不能接收服务器推送消息**，导致：
- 每次只能拉取一次数据，无法实时更新
- 刺激结果、时间轴帧无法自动推送到客户端

新版通过 NativeWebSocket 建立持久化双向 WebSocket 连接，所有问题根治。

## [2026-02-27] 综合优化 — 性能提升、模态加载修复、前端体验改善

### Bug 修复

#### fMRI/EEG 双模态加载修复（关键）
- **问题根因**: 上游训练管线将 fMRI（`hetero_graphs.pt`）和 EEG（`eeg_data.pt`）保存到两个独立文件中。`handle_load_cache` 每次只加载一个文件，导致用户加载 `eeg_data.pt` 时只看到 EEG、缺少 fMRI，反之亦然。
- **修复 1（代码 Bug）**: `_extract_time_series_both` 中 fMRI 提取路径缺少 `.x` 属性回退。EEG 路径已有 `getattr(g["eeg"], "x", None)` 兜底，但 fMRI 路径没有，导致旧格式 fMRI 数据静默丢失。现已对称补齐。
- **修复 2（功能增强）**: 新增 `_find_companion_cache` 方法：加载单模态文件后自动在同目录检测伴随文件（`hetero_graphs.pt` ↔ `eeg_data.pt`），若存在则自动合并，用户无需手动加载两个文件。

#### 前端 `stimPending` 永久卡死修复
- **问题**: 刺激请求发出后 WebSocket 断开，`stimPending = true` 永远不被重置，"施加刺激"按钮永久禁用。
- **修复**: 在 `ws.onerror` 和 `ws.onclose` 回调中调用 `_unlockStimBtn()`。

### 性能优化

#### `infer_ec_perturbation` — 批量推断（~7× 加速）
- 替换 N=200 次串行前向传播为分块批量推断（chunk_size=32）
- 7 次批量 PyTorch 调用替代 200 次串行调用，数值完全等价（max_err < 1e-8）

#### `ec_half_split_reliability._compute_ec` — 同上批量推断
- EC 可靠性验证中的有限差分计算同样使用分块批量推断

#### `infer_ec_jacobian` — lag 块求和向量化
- `J.reshape(N, n_lags, N).sum(axis=1)` 替代 Python 循环，消除 n_lags 次 (N,N) 矩阵加法的 Python 开销

#### `fit_surrogate` — NaN 替换向量化
- 列循环替换 NaN/Inf → 纯 NumPy 广播操作（无 Python 循环）

#### `compute_graph_metrics.local_eff` — 向量化（数值精确）
- 原 O(N) Python 循环（np.ix_ 子矩阵切片）→ 矩阵乘法 `diag(adj @ ec_abs * adj)` / n_nbrs²
- 数值完全等价（max_err < 1e-7），Pearson r = 1.000

### 健壮性改进

#### `_find_all_hetero_data` — 递归深度保护
